本文档为Big (1.0) 开发文档

Big是一个为了解决目前项目中越来越复杂的前端项目中而开发的一套完整的JavaScript库。

为了尽可能地优化网站项目后期的维护性，Big 引入、加载依赖于sea.js。

sea.js是一个遵循CommonJS(CMD)规范的JavaScript模块加载框架，可以实现JavaScript的模块化开发及加载机制。与jQuery等JavaScript框架不同，SeaJS不会扩展封装语言特性，而只是实现JavaScript的模块化及按模块加载。

Big代码执行高度依赖jQuery。

Big总共包含8种类型的模块，共包括Event(事件)、Utility(功能)、Collection(集合)、Library(方法)、View(功能)、Component(组件 [项目么有组件/公共组件] )、Extend(扩展)。

Event，事件集合，为Big自身提供的基础的用户事件方法；

Utility，功能集合，实现功能、获取/更新内、外部数据的方法集合；

Collection Big.coll，由Big自身内部获取的数据集合

Library Big.lib，常用的方法集合；

UI Big.ui，常用的UI、视图功能处理模块；

Component Big.com，已经构建好的组件，可直接调用；

Extend Big.ext，对Big做出的功能扩展，为第三方功能模块提供支持;

Cache Big.cache，页面上的事件/元素/方法缓存。

安装 (Install)

  • 引入sea.js文件；
  • 引入sea.config.js文件(seajs的配置文件)；
  • (如果需要sea可以加载css文件，则需引入sea.css.js文件，目前此文件已经与sea.js文件合并，无需另外引入)；
  • 使用sea.use方法引入页面文件所应用的js文件（一般推荐一个页面独立一个JS文件，不同的页面引用不同的JS文件）。

初始化 (Initialization)

因为Big依赖sea.js，而sea.js遵守CMD编码规范，所以页面文件统一要遵守CMD规范；

下面为sea的编码规范示例：CMD规范

// 所有模块都通过 define 来定义
define(function(require, exports, module) {
    var $ = require('jquery'),      // 引用jQuery
        Big = require('big')($[, options]);  // jQuery加载完成后，自动执行Big

    // 或者通过 module.exports 提供整个接口
    module.exports = {
        example : function() {
            alert('Big加载完成！');
            console.log(Big);
        }
    }

    $(function() {
        module.exports.example();
    });
})

在sea.js的加载方式中，Big需要两个参数，其中第一个参数为jQuery，第二个参数为可选项

(
    $ : jQuery,           // 必填，jQuery
    options : {           // 选填，选项
        onReady : function      // 选填，Big就绪时执行的回调
        config : json           // 选填，Big中的参数
    }
)

开发规范 (specification)

    方法参数规则（通常意义）

    • param json格式，方法中的必填参数
    • option json格式，方法中的可选选项
    • target jQuery选择器，控制(包含事件)/被控制(被事件改变)的元素
    • control jQuery选择器，控制某个事件的元素
    • content/column jQuery选择器，一个/多个被包含、处理的内容元素
    • handler function，必须存在的方法
    • callback function，可选的事件回调
    • onReady function，准备就绪时的回调
    • onChange function，改变时的回调
    • onChangeStart function，改变开始时的回调（通常会回传两个参数[当前的索引,下一个索引]）
    • onChangeEnd function，改变结束时的回调（通常会回传两个参数[当前的索引,上一个索引]）
    • onShow function，展开显示某个元素后的回调
    • onChangeEnd function，收起隐藏某个元素时的回调
    • wrapper jQuery选择器，容器
    • event string，事件名称
    • active number(index)，默认的索引
    • ease string(easing)，动画的缓动效果
    • delay number，某个计时器，通常作为一个循环的延时
    • time number，某个计时器
    • loop boolean，是否循环
    • autoplay boolean/millisecond，是否自动执行/自动播放的时间间隔
    • easing string(sting)，缓动效果
    • display boolean，是否显示
    • style string，样式名
    • width/height number，宽度/高度
    • prop json，属性，通常属性下标与原生JS成品属性名称一致，如果成员不存在，那么属性名称为自定义

    参数类型

    • number index/number/millisecond
    • string easing/style
    • json json
    • array array
    • function function
    • regex regex

    样式命名规范

    • 所有应该某个方法的元素，都应该为其加上特定的class名称，通常样式名称为“-xxx-xxx”
    • 内容包围盒子元素应该以-wrapper结尾
    • 内容包围主体元素应该以-container结尾
    • 内容容器盒子元素应该以-content结尾
    • 控制元素应该以-nav结尾，前后切换按钮应该另加上'prev-btn'/'next-btn'
    • 分页容器元素应该以-pagination结尾

    代码书写规范

    • 页面样式名遵守驼峰式规则即 className，js咐加的样式名遵守连接线规则即 class-name
    • 所有的var定义使用连续定义，定义分割使用","逗号，结束时加";"分号，如：var a = 1,b = 2,c = 3;
    • 所有的";"分号必须加上
    • 不能省略"{"、"}"，必须写上
    • 所有的运算符之间必须用" "空格格开
    • json下标尽量不要用双引号、单引号，json最后一个值后面不要带","逗号
    • jQuery元素请用$开头命名，如 var $box = $('.box');
    • 方法参数、数组等连续值请用" "空格格开，如：['1', '2', '3']、(param, options)
    • 经常使用的变量必须事先声明，方便调用存取

    注意事项

    • 不要人为添加任何全局变量，即定义变量时必使用var开头
    • 以大写字母开头的方法都需要使用 new 操作符进行构建
    • Big不支持外部调取，如有特殊需要，请在CMD中使用window.Big = Big，以达到外部调取
